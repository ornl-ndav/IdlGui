Using the Batch Mode

Using this GUI, it is possible to run live or in the background a set of Data Reduction.

The table is populated each time a data reduction has been run with success. It is then possible to modify the DATA and/or NORMALIZATION runs from the current selected row and re-run all the ACTIVE row or save it for later use.

The following picture shows the various states of a row:

  • 1st row is the freshly added data reduction process
  • 2nd row is an activated command line with no normalization run
  • 3rd row is an inactivated command line. This means that this command line won't be run
  • 4th and 5th row are activated run with both Data and Normalization runs

Once a row has been selected, each column of information is displayed in the Selected Run Number Information Box. The user can then change the active status of the row, the Data runs and if any, the Normalization runs. You can either click ENTER to validate the changes or wait for the application to pop up a window that confirm the changes when you select another row or click the REPOPULATE GUI button.


It is possible to move any selection up using the MOVE UP SELECTION and down using the MOVE DOWN SELECTION buttons. The it is also possible to delete the current selected row using DELETE SELECTION or all the active rows using DELETE ACTIVE.
Then to run all the active rows live, just click RUN ACTIVE LIVE, or to run them in the background, use RUN ACTIVE IN BACKGROUND. Running the jobs in background give you the control of the application back right away and you don't have to wait for the jobs to end.

If you are running the jobs live, a progress bar will keep you inform of the status of the jobs.
If you are running the jobs in background, the following web page (https://portal.sns.gov/snsapps/tabs/squeue.php?view=all) will give you the status of each jobs (row) you launched.
